## 技术方案：统一增强主题与作者推荐的引用紫色

### 修订摘要
- 用户明确“不要求完全一样的紫色”：目标从复制 Logo 主色调整为以 Codex 紫为视觉锚点、重新设计一套更适合深色正文阅读的紫色主题。
- 用户要求“新紫色也要有自己的边框颜色”：边框不再使用文字主色的同 RGB 透明版本，改用独立浅薰衣草紫 `rgba(196, 181, 253, 0.32)`。
- 色板改为分层设计：行内文字 `#C084FC`、深紫背景 `rgba(88, 28, 135, 0.18)`、独立浅紫边框、引用块边框 `#A78BFA`、引用块背景 `rgba(76, 29, 149, 0.14)`；测试由“同 RGB 派生”改为逐字段精确校验。
- 验证期视觉反馈认为首版紫色偏浅，并进一步给出湖人“紫金”参考：最终放弃偏粉玫紫，转为以经典深皇家紫 `#552583` 为基底的暗色适配，使用行内文字 `#A879D1`、背景 `rgba(85, 37, 131, 0.30)`、独立边框 `rgba(139, 101, 176, 0.58)`、引用块边框 `#74509A`、引用块背景 `rgba(57, 28, 88, 0.28)`；现有琥珀金标题/加粗色继续承担“金”色角色。

### 需求解析
- **目标**：消除增强主题中行内引用/代码的玫红色，设计一套与 Codex 品牌气质协调、但拥有独立文字色、背景色和边框色层次的深色主题紫。
- **输入**：用户提供的当前 ChatGPT Codex 深色界面截图，以及“不要求完全一样、要基于审美设计、边框要有自己的颜色”的修订要求。
- **输出**：改代码；更新增强主题默认值与 `data/author-config.json` 作者推荐值为同一套分层紫色色板，并补充可重复执行的逐字段配置回归验证。
- **边界**：不直接复制 Logo 像素色；不把同一 RGB 仅靠透明度机械复用于文字、背景和边框；不修改应用包体、用户数据、宽屏布局、输入增强及标题金色、加粗金色、链接蓝色等无关配色。

### 现状
- **相关代码 / 页面 / 接口 / 模块**：主题色由 `inject-wide-layout.mjs` 的 `DEFAULT_THEME_ENHANCEMENT_COLORS` 定义，用户配置通过 `buildOptions` 覆盖默认色板，最终写入 CSS 变量并作用于主会话 Markdown 的行内代码和引用块；作者推荐值位于 `data/author-config.json`，当前本机 `~/.codex-app-extension/config.json` 是指向该文件的软链接。
- **当前实现方式**：内置默认值与作者配置都以 `#df3079` 为主色，并用同一 RGB 的 `0.10`、`0.18`、`0.06` 透明度派生行内代码背景、边框和引用块背景；生成 CSS 后，行内代码消费 text/background/border 三个变量，引用块消费 text/background/border 三个变量。
- **现有问题 / 缺口**：截图中的 `ec-workflow` 行内引用呈玫红色，与整体 Codex 紫色气质不协调；旧色板还把文字、背景和边框全部绑定到同一个玫红 RGB，缺少边框与背景的独立明度层次。截图采样得到的 Logo 主色 `#B16FFF` 只作为色相锚点，不作为新主题的直接复制值。现有默认值、作者配置、README 示例和预览页仍共同固化旧玫红色，`verify.sh` 也没有校验分层色板一致性。
- **证据**：`inject-wide-layout.mjs:40-47`（旧默认色板）、`inject-wide-layout.mjs:622-625`（配置覆盖链）、`inject-wide-layout.mjs:1248-1253` 与 `inject-wide-layout.mjs:3109-3114`（CSS 变量写入）、`inject-wide-layout.mjs:3155-3174`（行内代码/引用块消费者）、`data/author-config.json:19-26`（作者推荐旧色板）、`README.md:143-149` 与 `README.md:184-193`（公开配置示例和字段语义）、`verify.sh:64-109`（现有生成 CSS 验证范围）。

### 修改方案
- **总体改法**：参考湖人紫金的“深皇家紫 + 金色强调”关系：保留现有 `#F2C94C` 标题/加粗金色，以 `#552583` 为紫色基底；行内文字提升到可读的 `#A879D1`，背景使用 `rgba(85, 37, 131, 0.30)`，独立边框使用 `rgba(139, 101, 176, 0.58)`；引用块使用更沉稳的 `#74509A` 与 `rgba(57, 28, 88, 0.28)`。
- **后端改动**：不涉及。
- **前端改动**：更新内置 `DEFAULT_THEME_ENHANCEMENT_COLORS`、作者推荐 `themeEnhancementColors` 和人工预览；行内代码的文字、背景、边框使用三个独立但协调的紫色值，引用块使用偏冷的结构紫与更深的块背景，引用正文颜色保持不变。
- **兼容处理**：不新增、不删除配置字段；既有自定义普通配置继续按配置优先级覆盖新默认值。当前配置为作者配置软链接，因此更新 `data/author-config.json` 后无需另写用户目录。README 示例与回归断言同时更新。
- **风险点**：边框过亮会抢过文字，背景过饱和会形成色块噪声；逐字段色值必须在默认配置、作者配置、生成 CSS、README 和预览中保持一致。

### 风险与注意事项
- 湖人深紫只作为背景和色相基底，不能直接用于暗色界面小字号文字；派生文字紫 `#A879D1` 在深紫行内背景上的对比度约为 `4.82:1`，兼顾整体收深与基本可读性。
- 独立边框使用较文字更低明度的紫色并提高到 58% 透明度，以稳定轮廓而不发亮；现有 `#F2C94C` 金色不扩散到代码边框，避免行内元素过度“队服化”。
- 保留 `blockquoteText` 的现有中性白透明色，避免把整段引用正文改成高饱和紫而降低长文可读性。
- 仓库存在用户已有的 Easy Coding 初始化改动，本任务只修改改动范围表列出的项目文件和本任务流程产物，不覆盖或清理其他变更。
