# 布局焦点环修复与 IME 防护收紧

## 任务概述

修复 Codex App 主对话框偶发蓝色焦点边框视觉问题，并收紧中文输入法 Enter 防护逻辑，避免普通组合输入键被误判为 Enter。

## 技术方案摘要

- 新增 `layoutFocusRingFix` 配置项，默认开启。
- 通过运行时 CSS 只隐藏顶层布局容器的误触焦点框，不影响输入框、按钮、菜单、链接等真实控件的焦点样式。
- 新增 CLI 与环境变量覆盖入口，保持“内置默认值 < 配置文件 < 环境变量 < CLI 参数”的既有优先级。
- 新增诊断字段，便于确认布局焦点环修复是否已生效。
- 将 IME guard 的 Enter 判断收紧为真实 Enter 信号，不再把 `keyCode=229` 普通组合键单独当作 Enter。

## 变更内容

- `inject-wide-layout.mjs`
  - 新增默认配置 `layoutFocusRingFix: true`。
  - 新增 `--enable-layout-focus-ring-fix` / `--disable-layout-focus-ring-fix`。
  - 新增 `CODEX_APP_EXTENSION_LAYOUT_FOCUS_RING_FIX` 环境变量读取。
  - 新增页面根节点状态 `data-codex-app-extension-layout-focus-ring-fix`。
  - 新增诊断输出 `layoutFocusRingFixEnabled`、`layoutFocusRingFixAttribute`、`layoutFocusRingFixInstalled`。
  - 新增 CSS 规则，仅处理 `main.main-surface`、`.app-shell-main-content-viewport`、`[data-app-shell-main-content-layout]`、`.thread-scroll-container`。
  - 移除 IME guard 中 `(event.keyCode === 229 && imeManagedKey)` 对 Enter 的误判。
- `README.md`
  - 同步新增能力说明、默认配置、字段说明、环境变量示例、CLI 示例和诊断字段说明。

## 技术决策

- 该问题判断为 Codex App 原生焦点态被触发，扩展的宽屏/全屏布局增强会放大其视觉影响。
- 修复策略定位为扩展侧兼容性视觉补丁，不全局禁用焦点样式。
- 保留真实交互控件的焦点提示，避免影响键盘可用性和可访问性。
