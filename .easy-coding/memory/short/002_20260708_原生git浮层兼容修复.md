# 原生 git 浮层兼容修复

## 任务摘要

- 目标：修复 Codex 升级后，开启宽屏增强时输入框上方残留不完整 git/diff 组件的问题。
- 范围：`inject-wide-layout.mjs` 的右侧 floating rail 识别、原生浮层变量隔离、宽屏变量 scope 写入、诊断输出，以及 `README.md` 诊断说明。
- 结果：已完成。当前实例已重注入，诊断确认右侧原生 `thread-floating-content` 继续参与主区域避让测量，但不再作为 `right-floating-rail-scope` 写入宽屏变量；composer 上方 / `thread-floating-content` 内的原生浮层会被标记为 `data-codex-app-extension-native-floating-panel="true"`，在自身子树内重置宽屏变量和横向偏移。
- 关键约束：不修改 Codex App 包体和用户数据；保留宽屏避让能力；保持增强可诊断、可回退。

## 技术记忆候选

> 只记录未来可能复用的工程事实。无则写“无”。

- 架构 / 接口决策：右侧 `thread-floating-content` / composer 上方原生浮层应作为主区域避让测量输入，但不能作为独立宽屏变量 scope 写入目标；这些原生浮层还需要本地隔离宽屏变量，避免继承根节点的宽屏 `max-width` / `translate`。
- 工程规则 / 工作流：新增诊断字段或兼容语义后必须同步 README；当前 Codex App 内置可用 Node 路径为 `/Applications/Codex.app/Contents/Resources/cua_node/bin/node`。
- 实现模式 / 复用写法：通过 `nativeFloatingPanel` / `useAsLayoutScope` 在诊断里显式暴露 rail 判定结果；运行时结构化浮层可通过 `thread-floating-content`、`bottom-full`、`diff-stat`、`repoAndDiffStats` 等面板壳识别，git/diff 文本再通过小型可交互候选中的 `文件已更改`、`+N -N` 等文本信号兜底；实际被隔离的目标可看 `nativeFloatingResetTargets`。运行时只扫描结构化浮层选择器和小型可交互文本候选，不对 `body *` 做全量文本扫描。
- 易错点 / 修复策略：不要把所有 absolute/fixed/sticky 右侧 panel 都追加为 `right-floating-rail-scope`；新版 Codex 原生环境/来源/git 面板若被写入或继承宽屏 `--thread-content-max-width` / `--thread-composer-max-width` / `--codex-app-extension-content-offset-x`，容易把 diff chip 压窄、错位或保留半截可点击区域。
- 验证经验：`./inject-current.sh --diagnose` 只做诊断，不安装新脚本；验证注入变更必须先运行 `./inject-current.sh`，再运行 `./inject-current.sh --diagnose`。
