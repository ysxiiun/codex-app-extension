# Tab 制表符输入增强

- 日期：2026-05-12
- 类型：新功能
- 项目模式：迭代项目

## 任务概述

为 Codex App 输入框新增可配置的 Tab 制表符输入能力。能力内置默认关闭，但已在用户本机 `~/.codex-app-extension/config.json` 中开启，便于写入带缩进的文本。

## 技术方案摘要

- 新增 `tabIndentEnhancement` 配置项，内置默认值为 `false`。
- 配置链路保持“内置默认值 < 配置文件 < 环境变量 < CLI 参数”。
- 新增 `CODEX_APP_EXTENSION_TAB_INDENT_ENHANCEMENT` 环境变量。
- 新增 `--enable-tab-indent-enhancement` / `--disable-tab-indent-enhancement` CLI 参数。
- 注入页面上下文后，仅在识别到 Codex 主输入框或 Plan 回复框时接管普通 `Tab`，写入 `\t`。
- 保留 `Shift+Tab`、带修饰键的 Tab 与非输入区 Tab 的原生行为，避免破坏全局键盘导航。
- 新增诊断字段，确认 Tab 增强是否启用、安装及最近处理事件。

## 变更内容

- `inject-wide-layout.mjs`
  - 新增默认配置、配置文件解析、环境变量读取、CLI 参数解析与 help 文案。
  - 新增 `isTabKey`、`insertTextareaTab`、`insertContentEditableTab`。
  - 新增 `installTabIndentEnhancement`，复用现有输入框识别逻辑。
  - 新增诊断输出 `tabIndentEnhancementEnabled`、`tabIndentEnhancementInstalled`、`tabIndentEnhancementState`。
- `README.md`
  - 新增能力说明、默认配置示例、字段说明、环境变量示例、CLI 示例和诊断字段说明。
- `~/.codex-app-extension/config.json`
  - 新增 `"tabIndentEnhancement": true`，只开启用户本机配置，不改变内置默认值。

## 验证结果

- `node --check inject-wide-layout.mjs` 通过。
- `git diff --check` 通过。
- 本机配置解析通过，`tabIndentEnhancement=true`。
- `./inject-current.sh` 重注入当前 Codex 实例成功。
- `./inject-current.sh --diagnose` 确认：
  - `tabIndentEnhancementEnabled=true`
  - `tabIndentEnhancementInstalled=true`

## 技术决策

- Tab 是系统焦点导航键，因此增强逻辑只在已识别的 Codex 输入框内接管普通 Tab。
- ProseMirror 输入框优先使用浏览器输入命令写入制表符，textarea 使用受控 value setter 和 input event，尽量兼容 Codex 页面状态管理。
- 能力默认关闭，符合输入行为增强的保守默认；用户本机通过配置文件显式开启。

## 未尽事项

- 未自动模拟 Tab 输入，避免向用户当前输入框写入测试字符。后续如需精确确认插入效果，可由用户在输入框中手动按 Tab 验证。
