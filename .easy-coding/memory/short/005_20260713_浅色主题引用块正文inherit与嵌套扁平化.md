# 浅色主题引用块正文改 inherit 与嵌套引用扁平化

## 任务摘要

- 目标：修复 Markdown 主题增强下引用块正文写死半透明白色（浅色主题对比约 1.59:1、几乎不可读），并消除嵌套引用重复叠加背景/边框/间距形成的二层灰块。
- 范围：`inject-wide-layout.mjs` 的默认色板与 blockquote CSS 规则、`verify.sh` 生成 CSS 回归断言、`README.md` 字段与默认值说明、`data/author-config.json` 作者推荐配置。
- 结果：已完成代码改动并通过离线硬门禁。引用正文默认色由白色改为 `inherit`（深/浅色自动跟随主题原生正文色，无需主题识别）；新增 `:where(blockquote) blockquote` 扁平化复位。用户判定验证完成并推进归档。
- 关键约束：不改深色既有视觉；不引入主题识别纯函数或主题变体属性；保留 `blockquoteText` 字段可配置；保持编码（mjs/README UTF-8，verify.sh/author-config US-ASCII）；只动 4 个真实项目文件。

## 技术记忆候选

> 只记录未来可能复用的工程事实。无则写“无”。

- 架构 / 接口决策：引用正文颜色改由 CSS `inherit` 承载（默认色板 `DEFAULT_THEME_ENHANCEMENT_COLORS.blockquoteText` → `DEFAULT_CONFIG` → 变量 `--codex-app-extension-theme-blockquote-text` → `:where(blockquote)` 的 `color`），避免引入主题识别逻辑即可同时适配浅/深色。
- 实现模式 / 复用写法：嵌套引用扁平化用 `… main.main-surface :where(blockquote) blockquote`——尾部真实元素选择器把 specificity 提到 (0,3,3)，高于基础 `:where(blockquote)` 的 (0,3,2)，同为 `!important` 时高特异性胜出；只复位 `background/border-left/border-radius/margin-inline/padding`，不写 `color` 以保持继承。
- 工程规则 / 工作流：`assertCssValue`（`inject-wide-layout.mjs:741-745`，经 `assertCssColor` 委托）只拒绝空值与 `;{}<>`，允许 `inherit`，故把默认值改为 `inherit` 不会破坏 `--configure`/install 校验；用户可见默认值变化需同步 README 与作者推荐配置。
- 易错点 / 修复策略：**CDP target 选择陷阱**——ChatGPT 开放调试端口时 `/json/list` 可能同时有多个 title 均为 “Codex” 的 page target；avatar-overlay 悬浮窗与主工作区 `app://-/index.html` 在 `selectTarget` 打分相同（`/codex/i`+100、`^app:`+60=160），平局按列表顺序取第一个，overlay 常排在前被选中，导致 surface 判为 unsupported、增强 enabled 但 installed=false。`--target` 用 `includes` 匹配，主窗口 url 是 overlay url 的子串，无法正向排除 overlay；要注入主工作区需让 overlay target 先消失（关闭悬浮层）或直连主工作区 ws。另：`inject-current.sh:115` 的 “Re-injecting…” 是静态横幅，`--diagnose` 实为只读诊断。
- 验证经验：`./verify.sh` 为项目唯一可执行测试基线，可静态断言生成 CSS 不变量；生成 CSS 断言可用 `css.match(/… :where\(blockquote\) blockquote \{([^}]*)\}/)`（JS 正则 `[^}]*` 跨行捕获规则块），顶层正则 `:where\(blockquote\) \{` 不会误匹配嵌套的 ` blockquote {`。有端口时 `CODEX_APP_EXTENSION_VERIFY_LIVE=1 ./verify.sh` 可做只读在线诊断，但需注意 target 选择可能落到 overlay。
