## 技术记忆候选

> 只记录未来可能复用的工程事实。无则写“无”。

- 架构 / 接口决策：右侧侧边栏开合属于运行时 DOM 布局变化，应在注入脚本内做 observer 调度稳定化，而不是修改 Codex App 包体。
- 工程规则 / 工作流：Easy Coding schema-2 迁移后，长期记忆为 `long/MEMORY.md` 索引 + `BUSINESS.md` / `TECHNICAL.md` 正文，旧版短期记忆已一次性沉淀并删除。
- 实现模式 / 复用写法：`window.resize` 保持即时刷新；`ResizeObserver` / `MutationObserver` 触发的布局刷新走短延迟合并，避免跟随侧栏动画逐帧重算。当前参数为首次稳定刷新 `80ms`，收尾刷新 `160ms / 320ms`。
- 易错点 / 修复策略：不要在每次宽屏刷新时全量 `clearWideLayoutVariables()` 再写回变量，这会制造 style mutation 并放大 ResizeObserver 反馈；应只清理过期变量目标。
- 验证经验：修改 `inject-wide-layout.mjs` 后至少执行 `node --check inject-wide-layout.mjs`、`git diff --check`，能连接当前 Codex 时执行 `./inject-current.sh` 和 `./inject-current.sh --diagnose`。
