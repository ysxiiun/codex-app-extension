## 技术记忆候选

> 只记录未来可能复用的工程事实。无则写“无”。

- 架构 / 接口决策：左侧栏不属于主会话内容区，应在 `.app-shell-left-panel` 内隔离 `--thread-content-max-width`、`--thread-composer-max-width`、`--markdown-wide-block-max-width`、`--codex-app-extension-content-offset-x` 与横向 padding，避免主内容宽屏变量影响侧栏。
- 工程规则 / 工作流：新增诊断字段时同步 README；运行时视觉问题应先通过 `--diagnose` 或 CDP 探针确认真实 DOM / computed style，再写 CSS 适配。
- 实现模式 / 复用写法：针对 Codex 左侧项目行，`overflow-x:hidden` 会让 computed `overflow-y` 变成 `auto`，若行内高度临界可能显示每行滚动条；可在 `.app-shell-left-panel [class*="group/folder-row"]` 下显式设置 `overflow-y:hidden`。
- 易错点 / 修复策略：项目行尾部 hover 操作区即使透明也可能静态占宽，导致标题区域提前缩短；应把该操作区绝对定位，默认 `pointer-events:none`，hover / focus-within 时恢复交互，并给内容区保留右侧 padding。
- 验证经验：左侧栏修复可用 CDP 抽样确认 `leftSidebar.contentOffsetX=0px`、`threadContentMaxWidth=100%`、项目行 `overflowY=hidden`、项目内容子容器宽度接近整行宽度。
