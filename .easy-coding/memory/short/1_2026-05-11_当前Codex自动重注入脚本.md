# 当前 Codex 自动重注入脚本

- 日期：2026-05-11
- 类型：新功能
- 项目模式：迭代项目

## 任务概述

新增一个用于配置修改后快速生效的脚本入口，自动发现当前正在运行且暴露远程调试端口的 Codex App，并复用现有 `inject-wide-layout.mjs` 完成重新注入。

## 技术方案摘要

- 新增 `inject-current.sh`，不启动新的 Codex App，只对当前运行实例执行重注入。
- 端口选择顺序：`--port`、`CODEX_APP_EXTENSION_PORT` / `CODEX_WIDE_PORT`、`lsof` 发现的 Codex 监听端口、默认 `9229`。
- 端口发现只作为候选来源，最终以 `http://127.0.0.1:{port}/json/version` 校验远程调试端口是否可用。
- 同步更新 README，说明配置文件修改后的重新生效方式和诊断命令。

## 变更内容

- `inject-current.sh`
  - 新增 Node 运行时检查，找不到系统 Node 时回退到 Codex App 内置 Node。
  - 新增当前 Codex 监听端口发现与去重逻辑。
  - 新增调试端口可用性检查和错误提示。
  - 调用 `inject-wide-layout.mjs` 复用既有注入能力。
- `README.md`
  - 新增“配置修改后重新生效”章节。
  - 补充 `inject-current.sh` 与 `inject-current.sh --diagnose` 使用示例。

## 问题与解决方案

- 问题：macOS 自带 Bash 在 `set -u` 下遍历空数组会触发 `unbound variable`。
- 解决：在遍历 `CANDIDATE_PORTS` 前先判断数组长度，保持脚本兼容性。

## 技术决策

- 不新增 Node.js 逻辑，不改现有注入脚本，降低影响面。
- 不自动启动 Codex，避免把“配置重载”入口和“启动入口”混在一起。
- 保留旧端口环境变量 `CODEX_WIDE_PORT` 的兼容性。

## 验证

- `bash -n inject-current.sh` 通过。
- `./inject-current.sh --diagnose` 通过，成功发现当前 Codex 调试端口 `9229`，目标为 `Codex / app://-/index.html?hostId=local`，返回 `ok: true`。
