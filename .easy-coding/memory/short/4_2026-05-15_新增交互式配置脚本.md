# 新增交互式配置脚本

## 任务概述

- 任务类型：新功能
- 目标：解决插件升级后旧 `~/.codex-app-extension/config.json` 缺少新增配置项，导致用户不易发现和配置新能力的问题。
- 范围：配置读取脚本、配置入口脚本、README 使用说明。

## 技术方案摘要

- 方案标题：新增交互式 config 脚本
- 核心改法：在 `inject-wide-layout.mjs` 中新增 `--configure` 配置模式，并新增 `config.sh` 作为用户入口，读取当前配置后逐项确认常用值和开关，最后写回完整配置结构。
- 关键约束：保留现有配置优先级；不在配置脚本中逐项编辑复杂主题色板；保留未知配置字段；配置模式不得连接 Codex 调试端口或执行注入。

## 变更内容

| 文件 / 模块 | 变更类型 | 说明 |
|---|---|---|
| `inject-wide-layout.mjs` | 修改 | 新增 `--configure` 分支、交互式配置流程、全量配置补齐、未知字段保留、管道输入兼容和 `y/n` 布尔解析；合并最新 master 后同步纳入 `tabIndentEnhancement` 新配置项。 |
| `config.sh` | 新增 | 新增配置脚本入口，复用现有 Node.js fallback 逻辑调用 `inject-wide-layout.mjs --configure`。 |
| `README.md` | 修改 | 新增配置脚本说明、全量补齐行为、复杂主题配置手动编辑提示和配置后重注入说明。 |

## 验证结果

- 自动化验证：`node --check inject-wide-layout.mjs`、`bash -n config.sh`、`git diff --check` 均通过。
- 人工验收：使用临时 `HOME` 模拟旧配置，验证脚本可展示当前值、切换布尔项、补齐 `layoutFocusRingFix` / `themeEnhancement` / 主题复杂配置块，并保留未知字段。
- 未验证项：真实 Codex 注入效果不属于本功能必需项；真实用户配置文件未被测试改动。

## 问题与解决方案

- 问题：管道输入测试时，`readline.question` 在多轮问题中可能提前结束，导致配置未写回。
- 解决：配置模式增加非 TTY 输入处理，先读取 stdin 行再逐项消费，兼容真实交互和自动化验证。
- 问题：布尔提示使用 `Y/n` / `y/N`，但原解析函数不接受单字符 `y` / `n`。
- 解决：扩展 `parseBooleanOption` 支持 `y` / `n`，并更新错误提示。

## 技术决策

- 将配置模式放在 `inject-wide-layout.mjs` 内部，复用现有 `DEFAULT_CONFIG`、校验函数和 JSON 写入路径，避免默认配置重复维护。
- `config.sh` 只做入口包装，不复制配置逻辑，降低后续维护成本。
- 复杂主题配置只补齐结构并提示用户编辑 JSON，避免在交互脚本中维护大量颜色和排版细节。
